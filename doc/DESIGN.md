# CloudRent 设计规范

> 适用范围：`frontend` 新前端。本文约束视觉、组件复用、交互动效和异常状态表现。

## 1. 主色调

CloudRent 的视觉目标是“安静、清晰、可信赖的运营后台”，不是营销页。界面应服务于高频扫描、批量处理和低误操作率。

### 1.1 色彩基线

| 用途 | 色值 | 使用位置 |
|---|---|---|
| 品牌蓝 | `#3B83F5` | 主按钮、当前菜单、重点数据 |
| 品牌紫 | `#7368F2` | 渐变过渡、权限/角色类操作 |
| 成功绿 | `#16A34A` | 已完成、已归还、正常状态 |
| 警告橙 | `#F59E0B` | 待处理、临期、提醒 |
| 危险红 | `#F24D5C` | 删除、黑名单、异常 |
| 信息青 | `#009999` | 权限、辅助标识 |
| 主文本 | `#172033` | 标题和重要字段 |
| 次文本 | `#667085` | 描述、表格辅助信息 |
| 边框 | `rgba(148, 163, 184, 0.28)` | 卡片、表格、输入框边界 |

### 1.2 背景和玻璃效果

- 页面背景使用浅色冷调渐变，允许少量柔和光感，但不得使用大面积单一紫蓝背景。
- 主内容区使用半透明白色玻璃卡：`bg-white/90` 到 `bg-white/95`，配合 `backdrop-blur` 和柔和阴影。
- 卡片圆角保持克制，常规卡片 16-20px，表格内元素 8-12px。
- 不使用装饰性渐变球、漂浮光斑或无业务意义的大面积插画。

### 1.3 状态色

- 车辆状态：未租用蓝/绿、已租用橙、维保红/橙、自用灰蓝。
- 订单状态：进行中蓝、已归还绿、逾期红、押金已返还青绿。
- 客户状态：白名单绿、黑名单红。
- 违章状态：未处理橙/红、已处理绿。
- 保养状态：已完成绿、待保养橙。

## 2. 组件复用规范

优先复用已有组件，不为单一页面新增重复 UI。

### 2.1 页面骨架

- `AppLayout.vue`：全局侧边栏、顶部区域和内容出口。
- `PageHeader.vue`：页面标题、说明和右侧主操作区。
- `GlassCard.vue`：所有主卡片容器。
- `StatCard.vue`：页面顶部统计卡片。

页面结构建议：

```text
PageHeader
StatCard grid
FilterBar
GlassCard(table/list)
FloatingPagination
Drawer/Dialog
```

### 2.2 查询和按钮

- 筛选区统一使用 `FilterBar.vue` + `FilterField.vue`。
- 搜索、重置、新增、批量删除等紧邻筛选栏放置，不分散到页面右上角。
- 小型操作统一使用 `CircleIconButton.vue`，只展示图标，依赖 `title` 提供含义。
- 批量删除使用 `BatchDeleteButton.vue`。
- 表格首列选择列不改动，其他列标题和内容默认居中。

### 2.3 表格和分页

- 所有分页页面使用 `FloatingPagination.vue`。
- 分页控件与上方卡片保持紧凑留白，不贴边，不大面积空白。
- 刷新后分页大小必须稳定，由页面状态显式维护。
- 总览统计不随当前页数据变化，应基于全量接口或 store 中全量统计。

### 2.4 表单

- 新增/编辑使用抽屉或弹窗，不跳离当前列表。
- 上传统一走 `/rental/oss/upload`。
- 颜色字段使用颜色选择器，展示时用真实颜色块和编码。
- 日期时间字段必须和后端格式兼容，不在页面内临时拼接不可解析格式。

## 3. 关键交互动效

动效只用于反馈状态和降低认知成本，不做装饰性动画。

### 3.1 页面切换

- 路由切换保持轻量淡入，不使用大位移动画。
- 财务首页、日报、月报切换后必须重新加载图表数据。
- 系统管理 Tab 切换时保留当前页面上下文，不刷新整页。

### 3.2 控件反馈

- 图标按钮 hover：轻微提升背景透明度和阴影。
- 主要按钮 hover：颜色加深或轻微上浮，位移不超过 2px。
- 表格行 hover：使用浅色底，不改变行高。
- 危险操作 hover 只能强化危险色，不使用夸张动画。

### 3.3 图表

- ECharts 使用系统风格的细线、柔和面积渐变和圆角柱。
- 图表 toolbox 不使用框架原生硬边样式；需要导出/还原时使用系统风格的小型图标按钮。
- tooltip 使用半透明白底、细边框、圆角和清晰阴影。

## 4. 错误、加载中、空数据状态

### 4.1 加载中

- 表格加载使用 Element Plus `v-loading` 或局部 skeleton，不遮挡整个应用。
- 图表加载时显示轻量占位，不显示假数据。
- 按钮提交中要禁用当前按钮，避免重复提交。

### 4.2 空数据

- 空表格显示简短业务文案，例如“暂无车辆数据”“暂无待处理违章”。
- 空态不使用大插画，不占用过多垂直空间。
- 如果当前空态由筛选导致，应保留“重置”入口。

### 4.3 错误状态

- 接口错误使用短消息提示，说明失败动作，例如“保存车辆失败”“权限不足，无法删除”。
- 删除、押金返还、角色权限分配等高风险操作必须二次确认。
- 权限不足不是系统异常，应使用明确提示并阻止后续请求。

### 4.4 表单校验

- 必填字段在提交前本地校验。
- 手机号、身份证、里程、金额、折扣、日期等字段必须做格式或范围校验。
- 业务校验优先复用后端已有接口，例如车牌存在校验、子部门/子菜单校验、角色绑定用户校验。

## 5. 禁止事项

- 不新增与现有风格冲突的大型卡片套卡片布局。
- 不把新增按钮分散到页面右上角，优先放在筛选栏附近。
- 不用英文作为面向用户的主按钮或状态文案。
- 不硬编码统计数据模拟真实业务结果。
- 不绕过已有权限组件和权限工具函数直接展示写操作。
- 不为了局部页面创建重复分页、重复搜索框或重复状态徽章。
