# 车辆租赁管理系统 — 产品需求文档（PRD）

> 版本：v1.0　|　更新日期：2026-06-25　|　文档状态：MVP 已实现

## 一、产品定位

面向**中小型汽车租赁公司**的内部运营管理系统（后台 Admin），覆盖从车辆建档、客户管理、车辆出租与归还、违章与保养，到财务统计的完整业务闭环。

- **产品形态**：B 端 Web 后台管理系统（前后端分离）。
- **目标用户**：租赁公司内部员工，按岗位分为系统管理员、运营专员、财务专员、维保专员、客服专员。
- **核心价值**：用一套系统替代线下台账与多张 Excel，实现车辆资产、订单、客户、财务数据的集中管理与实时统计，降低人工差错、提升结算与决策效率。

## 二、用户痛点

| 角色 | 痛点 | 系统解法 |
|---|---|---|
| 运营专员 | 线下记录车辆状态，易出现"一车两租"；租金按天手算易错 | 出租即锁定车辆状态为"已租"，归还自动按 `天数 × 日租 × 折扣` 算应付/实付 |
| 财务专员 | 收入靠月底翻台账汇总，无法实时掌握经营情况 | 日报/月报 echarts 图表 + 应收/实收/押金实时统计，支持数据导出 |
| 维保专员 | 不清楚哪些车该保养，保养记录散落 | 按行驶里程自动计算应保次数，列出"待保养车辆"，保养后实保次数自动 +1 |
| 客服专员 | 客户信息重复录入，黑名单客户难以拦截 | 按手机号查老客户自动回填；身份证号自动解析性别/年龄/生日；黑白名单标记 |
| 管理员 | 不同岗位权限难以隔离，越权操作风险 | RBAC 权限模型，菜单级 + 按钮级双重控制，数据权限隔离 |
| 风控 | 押金退还时遗漏未处理违章 | 押金返还前自动校验该车租期内是否有未处理违章，有则拦截 |

## 三、MVP 范围

### 已实现（本期）

1. **权限管理**：用户、角色、部门、菜单的增删改查；用户绑定角色、角色分配权限；菜单动态下发；菜单级 + 按钮级权限控制。
2. **数据初始**：车辆厂商、车辆品牌（厂商→品牌二级联动）、出租类型（折扣）、车辆信息（含 OSS 图片上传、多条件检索）。
3. **日常业务**：车辆出租（选车 → 选出租类型 → 录客户 → 生成订单）、车辆归还（算租金、改车辆状态）、订单详情（含押金返还 + 违章校验）、客户管理、违章处理、保养管理。
4. **财务统计**：日报 / 月报（应收/实收/押金 + 出租/归还 echarts 图表）、数据导出、定时邮件上报业绩。

### 暂不在本期范围（Future）

- 客户端（C 端）在线下单小程序 / App。
- 在线支付与第三方对账。
- 短信通知（当前仅邮件）。
- 多门店 / 多组织数据隔离的精细化运营。
- 报表的自定义时间区间与多维下钻。

## 五、核心流程

### 1. 车辆出租流程

```
检索未租车辆(status=0)
  → 点击「出租」弹出车辆信息(选出租类型/确认里程押金)
  → 下一步：录入客户(手机号查老客户回填 / 身份证号自动解析性别年龄生日)
  → 提交订单：新客户先入库拿 id → 创建订单
  → 后端：生成订单号、记录出租时间、车辆状态改为「已租」(status=1)
```

### 2. 车辆归还流程

```
检索未归还订单(return_time is null)
  → 点击「归还」弹窗(默认当前时间)
  → 校验：归还里程必须 > 起租里程
  → 自动结算：应付 = 向上取整天数 × 日租；实付 = 应付 × 折扣
  → 后端：更新订单归还信息、车辆状态改「未租」、按里程更新应保次数
```

### 3. 押金返还流程（风控）

```
订单详情 → 已归还订单点「押金返还」
  → 后端统计该车在本订单租期内是否有「未处理违章」
  → 有：提示前往违章管理处理，拦截退款
  → 无：押金返还状态置为「已返还」
```

### 4. 财务统计流程

```
财务日报/月报页面
  → 调用统计接口(按 rental_time/return_time 聚合)
  → 日报：当天各小时出租/归还数量 + 当日应收/实收/押金
  → 月报：当月各天出租/归还数量 + 当月汇总
  → echarts 渲染折线图，支持 line/bar 切换、导出图片
  → 定时任务每晚 20:00 汇总当日收入邮件上报
```

## 六、技术方案

### 整体架构

前后端分离，前端 SPA 通过 RESTful API 与后端交互；后端无状态，JWT + Redis 管理会话。

```
浏览器(Vue SPA :9527)  ──HTTP/JSON──>  Spring Boot(:8888)  ──>  MySQL 8 / Redis / 阿里云 OSS
        │                                      │
   动态路由菜单                          Spring Security + JWT 鉴权
```

### 后端

- **框架**：Spring Boot 3.0.10 / JDK 17
- **持久层**：MyBatis-Plus 3.5.5（分页插件、逻辑删除、自动填充、代码生成器）
- **安全**：Spring Security + JWT（jjwt 0.12.5），token 存 Redis，无状态会话；`@PreAuthorize` 方法级按钮权限
- **存储**：MySQL 8（库 `car_rental_system`）、Redis（token）、阿里云 OSS（图片）
- **其他**：Hutool、EasyExcel（导出）、spring-boot-starter-mail（定时邮件）、Swagger
- **分层**：controller / service(+impl) / mapper / entity / vo / config / security / utils

### 前端

- **框架**：Vue 2.6 + Element UI 2.13 + Vue Router 3 + Vuex 3（基于 vue-element-admin 模板）
- **构建**：Vue CLI 4
- **图表**：ECharts 4.2.1
- **请求**：Axios（统一封装 `utils/request.js`，token 放请求头）
- **权限**：登录后拉取 `/menuList` 动态挂载路由；`hasPermission` 指令控制按钮显隐

### RBAC 模型

`用户(sys_user) — 用户角色(sys_user_role) — 角色(sys_role) — 角色权限(sys_role_permission) — 权限(sys_permission)` + 部门(sys_dept)。权限按 `permission_type` 分为 0 目录 / 1 菜单 / 2 按钮。

### 关键设计

- **配置外置**：数据库密码、OSS 密钥、邮箱授权码均通过环境变量注入，不进代码库。
- **数据冗余**：违章、保养表冗余 `auto_num` 车牌字段，便于检索与展示。
- **金额取整**：租金天数向上取整（`Math.ceil`），实付按折扣 `parseInt` 取整。

## 七、主要风险

| 风险 | 影响 | 应对 |
|---|---|---|
| **敏感信息泄露**：OSS 密钥、邮箱授权码曾出现在配置中 | 密钥被盗用、产生费用 | 已改为环境变量注入；建议定期轮换密钥 |
| **布尔/数值字段映射**：性别、状态等 `Boolean` 字段与前端数值选项不匹配 | 编辑回显错误 | 已统一前后端类型；新增字段需注意一致性 |
| **车辆状态与订单一致性**：依赖业务代码维护，无数据库外键约束 | 可能出现状态与订单不符的脏数据 | 出租/归还在 service 层同步更新状态；建议加事务与定期校验 |
| **并发出租**：两人同时为同一辆车下单 | 一车两租 | 当前无锁；后续需加乐观锁 / 状态校验 |
| **财务统计依赖系统时间**：日报/月报按 `now()` 聚合 | 跨时区或测试数据时间不符则图表为空 | 统计口径已对齐；测试需保证数据落在当日/当月 |
| **定时邮件**：发件依赖第三方 SMTP 授权码 | 授权码失效则上报中断 | 授权码外置；建议加发送失败告警 |
| **无自动化测试**：当前以手动联调验证 | 回归成本高、易引入回归缺陷 | 后续补充单元测试与接口测试 |
