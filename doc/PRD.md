# CloudRent 车辆租赁管理系统 PRD

> 版本：v1.1 | 状态：MVP 已实现 | 更新日期：2026-06-30

## 1. 产品定位

CloudRent 是面向中小型汽车租赁公司的内部运营管理系统。系统服务于门店日常经营，不面向终端客户，核心目标是把车辆、客户、租约、押金、违章、保养和财务数据放在同一个后台里，减少纸质台账和 Excel 流转带来的错漏。

产品形态是 B 端 Web 管理后台，采用前后端分离架构。前端提供 iOS 风格的玻璃拟态管理界面，后端提供基于 RBAC 的业务接口、权限菜单、按钮权限和统计接口。

目标用户包括：

- 系统管理员：维护用户、角色、部门、菜单和权限。
- 运营专员：维护车辆资料、出租类型，处理出租和归还。
- 客服专员：维护客户资料，处理客户黑白名单。
- 维保专员：维护车辆保养记录，查看待保养车辆。
- 财务专员：查看收入、押金、订单趋势和财务日报/月报。
- 只读审计人员：查看经营数据，不允许写入业务数据。

## 2. 用户痛点

| 用户 | 痛点 | 产品解法 |
|---|---|---|
| 运营专员 | 车辆状态靠人工记录，容易出现一车多租 | 出租下单后自动锁定车辆状态，归还后恢复可租状态 |
| 运营专员 | 还车租金、折扣、天数靠手算，结算易错 | 归还时按完整时间差向上取整，自动计算应付和实付 |
| 客服专员 | 老客户重复录入，身份证信息手动填写成本高 | 按手机号查询老客户，身份证自动解析性别、年龄、生日 |
| 财务专员 | 收入和押金依赖月底汇总，无法实时掌握经营情况 | 财务首页、日报、月报展示实时统计和出租/归还趋势 |
| 维保专员 | 不知道哪些车已经达到保养周期 | 基于应保次数和实保次数展示待保养车辆 |
| 管理员 | 不同岗位权限边界不清，容易越权操作 | 菜单级和按钮级权限同时控制，前端只展示有权限的操作 |
| 审计人员 | 需要看数据但不能误操作 | 只读角色隐藏写按钮，并在前端请求层拦截写接口 |

## 3. 核心用户旅程

### 3.1 管理员配置权限

1. 管理员登录系统。
2. 在用户管理中创建用户，设置头像、部门、基础信息。
3. 在角色管理中创建角色，并打开权限分配面板。
4. 系统按 `sys_role_permission` 回显已有权限，管理员勾选菜单和按钮权限。
5. 用户重新登录后，侧边栏菜单和按钮操作按权限生效。

验收结果：无权限用户看不到对应菜单；只读用户不能新增、编辑、删除、处理、归还或返还押金。

### 3.2 运营完成一次出租

1. 运营进入汽车出租页面。
2. 通过厂商、品牌、车牌、租金、上牌日期等条件查询可租车辆。
3. 选择车辆，选择出租类型，录入客户手机号和身份证。
4. 系统查询老客户并回填；新客户先保存客户信息。
5. 提交订单后生成租约，车辆状态变为已租。

验收结果：自用、维保、已租车辆不会作为可租车辆提交；订单创建后车辆不再出现在可租列表。

### 3.3 运营完成一次归还

1. 运营进入车辆归还页面。
2. 页面使用未归还订单接口查询待还车辆。
3. 打开归还弹窗，默认归还时间为当前时间。
4. 录入归还里程，系统校验必须大于起租里程。
5. 系统计算应付、实付并提交完整订单上下文。
6. 后端更新订单、车辆状态和应保次数。

验收结果：同日归还至少按 1 天计费；跨时分秒归还按完整时间差向上取整。

### 3.4 财务查看经营表现

1. 财务进入财务首页查看总览。
2. 点击财务首页、财务日报、财务月报三个入口切换视图。
3. 日报展示当日收入、押金、出租趋势和归还趋势。
4. 月报展示当月收入、押金、出租趋势和归还趋势。
5. 保养费作为运营成本口径参与展示。

验收结果：从日报切到月报或从月报切到日报时，数据会重新加载，不依赖先回财务首页。

### 3.5 风控处理押金返还

1. 运营或财务进入订单总览。
2. 对已归还订单点击押金返还。
3. 系统调用违章统计接口，检查该车租期内是否存在未处理违章。
4. 有未处理违章时阻止返还；无违章时调用押金返还接口。

验收结果：押金返还必须先经过违章校验，前端不允许绕过该流程。

## 4. MVP 功能列表

### 权限与框架

- 登录、退出、token 持久化。
- 登录后获取 `/rental/auth/info` 和 `/rental/auth/menuList`。
- 按后端权限过滤侧边栏菜单，同时兼容旧前端路径。
- 系统管理旧路径可直达新前端 Tab 页面。
- 按按钮权限控制新增、编辑、删除、绑定角色、分配权限、出租、归还、押金返还等操作。

### 系统管理

- 用户分页查询、条件搜索、头像上传、增删改、绑定多角色。
- 部门树查询、后端条件搜索、删除前子部门校验、维护排序和上级部门。
- 角色分页查询、条件搜索、创建人对齐当前登录用户、删除前绑定用户校验、权限回显。
- 菜单树管理、上级菜单选择、删除前子节点校验、维护 `parentLabel`。

### 基础资料与车辆

- 厂商维护，支持排序字母 `orderLetter`。
- 品牌维护，按厂商和品牌名走服务端查询。
- 出租类型维护，支持分页、搜索、折扣展示。
- 车辆信息维护，支持厂商/品牌绑定、状态、颜色选择器、图片上传、租金和上牌日期范围查询。
- 车辆状态包含未租、已租、维保、自用。

### 出租与客户

- 可租车辆走后端状态条件查询。
- 老客户按手机号接口回填。
- 客户表单校验手机号、身份证、黑白名单状态。
- 出租流程保存客户并创建订单。

### 订单、违章与保养

- 订单总览使用后端分页和条件查询。
- 车辆归还使用未归还订单接口。
- 押金返还前调用未处理违章统计。
- 违章新增前校验车牌存在，日期筛选精确到秒。
- 保养新增车辆来源使用待保养车辆接口。
- 保养页面支持查看保养记录和待保养次数。

### 财务与首页

- 首页展示真实订单、车辆、客户和待处理违章数据。
- 财务首页展示总览、收入趋势、出租类型收入构成和运营成本。
- 财务日报展示当日成本、出租和归还趋势。
- 财务月报展示当月成本、出租和归还趋势。
- 图表样式与系统玻璃拟态风格一致。

## 5. 功能黑名单

以下能力不属于当前 MVP，不应在当前代码里临时拼接或伪实现：

- C 端下单小程序、App 或公开租车官网。
- 在线支付、第三方支付回调、银行对账。
- 电子合同、电子签名、OCR 证件识别。
- 多门店独立账套、跨门店调拨和门店库存隔离。
- 复杂审批流、工单系统、消息中心。
- 地图定位、车辆 IoT、实时 GPS 轨迹。
- 大屏驾驶舱、BI 自定义报表和任意维度下钻。
- 绕过后端权限模型的前端本地假权限。
- 为了展示效果硬编码业务统计数据。

## 6. 验收标准

### 权限验收

- 管理员能看到全部菜单和全部按钮。
- 只读审计角色只能查看数据，所有写按钮隐藏。
- 即使通过控制台触发写请求，前端请求层也会拦截无权限写接口。
- 角色权限弹窗打开时，已有权限必须按 `sys_role_permission` 正确勾选。

### 业务验收

- 新增车辆时厂商、品牌、颜色、图片、状态能正确保存和回显。
- 汽车出租只展示可租车辆，不展示已租、维保、自用车辆。
- 车辆归还时归还里程必须大于起租里程。
- 押金返还前必须校验租期内未处理违章。
- 保养页面能展示保养记录，也能筛选待保养车辆。

### 数据验收

- 所有分页页面刷新后页容量稳定，不退回全量列表。
- 总览卡片展示全量统计，不随当前分页或筛选变化。
- 搜索条件应优先走后端查询接口，不依赖前端全量过滤。
- 财务日报/月报切换后必须重新请求并刷新图表。

### 交付验收

- `frontend` 下执行 `npm run build` 通过。
- `backend` 下执行 `mvn -q compile` 通过。
- `start-cloudrent.command` 能启动前端、后端和 Redis，并在退出时清理本脚本启动的进程和缓存。
- 远端仓库不包含 `.gitignore` 文件。
